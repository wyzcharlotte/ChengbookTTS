## 项目概述

**ChengbookTTS v2.0.0** — 统一多模型 TTS 平台，集成 CosyVoice3、SoulX-Podcast、IndexTTS 三大模型，通过统一 API 提供语音合成服务。主要服务于"诚太乙"AI 教练系统。

## 架构

```
chengbook_tts/
├── engines/          # 引擎层 (base → cosyvoice3/soulxpodcast/indextts_v1/indextts_v2)
├── server/           # FastAPI 服务 (routes/, console.py, concurrency.py, history.py)
├── cli/              # 命令行工具
├── client/           # Python SDK
├── config/           # 配置 (settings.py, voices.py, emotions.py, model_configs/*.yaml)
├── vendor/           # 外部模型源码 (cosyvoice, matcha, soulxpodcast, indextts)
├── prompts/          # 音色提示音频
└── profiles/         # 用户偏好持久化
```

### 引擎能力矩阵

| 引擎 | 采样率 | 流式 | 情绪 | 语速控制 | 分词 |
|------|--------|------|------|----------|------|
| CosyVoice3 | 24000 | ✅ | ✅ | ✅ | ✅ |
| SoulX-Podcast | 24000 | ❌ | ✅(采样参数) | ❌ | ❌ |
| IndexTTS v1 | 24000 | ❌ | ❌ | ❌ | ❌ |
| IndexTTS v2 | 22050 | ❌ | ✅(8维向量) | ❌ | ❌ |

### API 端点

- `/api/health` — 健康检查 + 能力声明
- `/api/voices` / `/api/emotions` — 音色/情绪列表
- `/api/tts` — 非流式合成 → WAV
- `/api/tts/stream` — 流式合成 → PCM raw
- `/api/voices/custom` — 自定义音色上传/删除
- `/api/models` / `/api/model/switch` — 模型管理
- `/api/profile` — 用户偏好持久化
- `/api/history` — 合成记录展板
- `/v1/audio/speech` — OpenAI 兼容端点
- `/` — Web 控制台 (内联 SPA)

### 关键设计决策

1. **单例 ModelManager** — 同一时刻只有一个模型在 GPU 上，切换时自动卸载/加载
2. **预缓存** — CosyVoice3 启动时预计算所有音色特征和情绪 token
3. **能力声明** — 每个引擎声明 capability，前端据此灰显 UI
4. **情绪抽象** — 8 种通用情绪，各模型用不同机制实现
5. **OpenAI 兼容** — `/v1/audio/speech` 供诚太乙等外部产品调用
6. **并发控制** — asyncio.Semaphore + ThreadPoolExecutor，默认串行(GPU安全)
7. **自定义音色持久化** — custom_voices.json + WAV 文件，重启自动恢复

## 2026-07-06 (第二次会话): WebUI 转录文本输入框条件禁用

### 问题
WebUI 自定义音色上传区的"转录文本"输入框在所有模型下都可用，但实际上只有 SoulXPodcast 引擎会使用 `prompt_text`。CosyVoice3 的 `register_voice()` 虽然接受 `prompt_text` 参数但完全忽略它（只提取音频特征），IndexTTS v1/v2 也不使用。

### 改动
**console.py** (Web 控制台内联 SPA):

1. **CSS** (`input:disabled`): 新增禁用输入框样式 `opacity: 0.35; cursor: not-allowed; pointer-events: none`，与现有 `section-disabled` 风格一致。

2. **HTML**: 转录文本输入框下方新增警告提示 `<span id="promptTextNote">`，默认隐藏，非 SoulXPodcast 时显示 "⚠ 当前模型不支持转录文本，仅 SoulXPodcast 可用"。

3. **`updateCapUI()`**: 新增转录文本输入框的状态管理：
   - 检查 `activeModel === 'soulxpodcast'`
   - 不支持时：禁用输入框 + 更换 placeholder 为 "当前模型不支持转录文本（仅 SoulXPodcast 可用）" + 显示红色警告
   - 支持时：恢复可用状态 + 原始 placeholder + 隐藏警告

4. **`uploadVoice()`**: 仅在 `activeModel === 'soulxpodcast'` 时才将 `prompt_text` 附加到 FormData，防止无效数据发送到后端。

### 设计决策
选择**灰掉（disabled）**而非隐藏，原因：
- 与现有的 `section-disabled` 模式一致（情绪/语速/分词在不支持时也是灰掉整个卡片）
- 用户能看到功能存在，只是当前模型不支持
- 切换模型后自动恢复，无需重新发现功能
- 布局保持稳定
