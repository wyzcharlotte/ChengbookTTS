# 诚书记 TTS 接入诚太乙 — 配置指南

> 将诚书记 TTS 服务以 OpenAI 兼容方式接入诚太乙产品，实现 AI 教练模拟客户语音功能。
> **一次配置，后续所有音色/情绪切换均在网页控制台完成，无需再改 JSON。**
>
> 当前版本支持 **CosyVoice3**（轻量快速，支持流式）和 **SoulX-Podcast 1.7B**（多说话人对播）两个引擎，一键切换。

---

## 前提条件

1. 诚书记 TTS 服务已启动，端口 `8080`
2. 诚太乙与 TTS 服务网络互通（能 ping 通 `192.168.80.74`）

### 启动服务

```bash
# 默认 CosyVoice3 引擎
python -m chengbook_tts.cli serve --port 8080

# 指定引擎
python -m chengbook_tts.cli serve --model soulxpodcast --port 8080
```

---

## 新建 AI 配置

在诚太乙产品中打开「新建 AI 配置」页面，按以下内容逐项填写：

### 配置名称

```
诚书记 TTS
```

### 服务类型

选择：**`TTS (语音合成)`**

### 部署框架

选择：**`OpenAI`**

### 模型名称

```
tts-1
```

### 端点地址

```
http://192.168.80.74:8080/v1
```

> ⚠️ 注意：
> - 地址必须以 `/v1` 结尾，诚太乙会在其后自动拼接 `/audio/speech`
> - IP `192.168.80.74` 为 TTS 服务所在机器，如部署在其他机器请替换

### 配置参数 (JSON)

```json
{"voice":"alloy","response_format":"wav","speed":1.0}
```

> 💡 **这个 JSON 以后不需要再修改。** 所有音色、情绪、语速的切换通过下方控制台完成。
> - `voice`: 固定填 `alloy`，实际音色由控制台的 profile 决定
> - `response_format`: 推荐 `wav`（最通用，无需 ffmpeg），也支持 `mp3`
> - `speed`: 固定填 `1.0`，实际语速由控制台的 profile 决定

## voice 映射关系

诚太乙传入的 OpenAI voice 名称会自动映射到诚书记内部音色：

| 诚太乙 voice | 诚书记音色 | 说明 |
|-------------|-----------|------|
| `alloy`, `nova`, `shimmer` | `woman` | 女客户-小诚 |
| `echo`, `fable`, `onyx` | `man` | 男客户-老书 |

如果上传了自定义音色，诚太乙调用时仍使用上表中的 voice 值，实际音色由控制台的 profile 决定。

## 保存并接入

配置保存后，在诚太乙中触发一次 TTS 调用。服务端日志出现以下内容表示接入成功：

```
OpenAI TTS: voice=alloy→woman, fmt=wav, text_len=12, text=你好...
OpenAI TTS done: duration=2.10s, elapsed=0.85s
```

## 切换音色 / 情绪 / 语速

**使用 Web 控制台 — 不需要再进诚太乙改 JSON。**

在任意浏览器打开控制台地址：

```
http://192.168.80.74:8090/
```

控制台功能：

- **模型切换** — 在 CosyVoice3 和 SoulX-Podcast 之间一键切换，自动卸载旧模型
- **音色选择** — 点击按钮切换，预设音色（女客户/男客户）+ 自定义上传音色均可选
- **情绪选择** — 平和、开心、生气、悲伤、轻声、大声、急躁、困惑 8 种
- **语速调节** — 滑块 0.5x ~ 2.0x
- **分词开关** — 开启后二字词自动加 `[]`，改善中文断句准确性
- **自定义音色** — 上传 WAV 音频进行音色克隆，两个模型共用一个音色库
- **在线试听** — 输入文本直接合成播放，验证效果

点击任意按钮即刻生效，诚太乙下一次调用就会使用新配置。切换过程对诚太乙完全透明——它只看到同一个 API 地址和同一份 JSON。

## API 端点参考

| 端点 | 方法 | 说明 |
|------|------|------|
| `/v1/audio/speech` | POST | OpenAI 兼容 TTS（诚太乙调用此端点） |
| `/v1/models` | GET | 模型列表 |
| `/v1/health` | GET | 健康检查 |
| `/api/tts` | POST | 原生 TTS（支持 speed/segment） |
| `/api/tts/stream` | POST | 流式 TTS |
| `/api/voices` | GET | 音色列表 |
| `/api/emotions` | GET | 情绪列表 |
| `/api/voices/custom` | POST/DELETE | 自定义音色管理 |
| `/api/models` | GET | 可用模型列表 |
| `/api/model/switch` | POST | 切换模型 |
| `/api/profile` | GET/POST | 配置管理 |
| `/` | GET | Web 控制台 |

## 常见问题

| 问题 | 解决 |
|------|------|
| 诚太乙无法连接 | 检查 TTS 服务是否启动、网络是否互通（`ping 192.168.80.74`）、端点地址是否正确 |
| 返回 401 错误 | TTS 服务设置了 `API_KEY`，诚太乙的请求头需要携带 `Authorization: Bearer <KEY>` |
| 控制台无法打开 | 确认浏览器能访问 `http://192.168.80.74:8080/`，防火墙是否放行 8080 端口 |
| 切换模型后音色不见了 | 控制台会自动刷新，稍等几秒；自定义音色两个模型共享，切换后自动恢复 |
| 合成音质不佳 | 检查控制台当前选中的音色/情绪是否正确；可在线试听验证后保存 |
| 格式不支持 | `response_format` 推荐 `wav`（WAV 格式最通用），如需 mp3 需服务端安装 ffmpeg |
| 语速不生效 | 诚太乙 JSON 中的 `speed` 固定为 `1.0`，实际语速通过 Web 控制台滑块调节 |
